# Monitoring Stack

This document describes the monitoring components used in the Proxmox homelab.

## Components

### Prometheus
Prometheus is used as the main metrics collection system in the lab. It gathers metrics from infrastructure components and services.

### Prometheus PVE Exporter
The Proxmox VE exporter provides metrics from the Proxmox environment so they can be collected by Prometheus.

### Grafana
Grafana is used for dashboards and visualization of collected metrics. It provides a central place for observing system health and trends.

### Alertmanager
Alertmanager is used to manage alerts generated by Prometheus. It helps organize alert routing and notification logic.

### Blackbox Exporter
Blackbox Exporter is used for endpoint probing and availability checks. It is useful for verifying whether selected services are reachable.

## Goals of the Monitoring Stack

- observe service health
- collect infrastructure metrics
- visualize trends and status
- support troubleshooting
- improve operational awareness

## Practical Value

This monitoring stack is important for learning how to:
- monitor infrastructure proactively
- detect service problems
- visualize operational data
- build better documentation around system health
